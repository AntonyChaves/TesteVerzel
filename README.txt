Documento para uso da aplicação:

Na mesma página do documento estão as pastas de Back e Front, o BackEnd foi feito em Java e o FrontEnd em Javascript com React.

Cada um dos projetos tem sua forma de testar, o BackEnd está localizado no arquivo zip compactado com o nome 'BackEnd', para testar é necessário descompactar o arquivo e então irá aparecer a pasta do BackEnd do projeto, para testar é recomendado utilizar a IDE Intellij, nesta IDE, deve-se optar por abrir um projeto e selecionar a pasta descompactada do BackEnd, após isso, o usuário deverá ir abrindo as páginas do projeto até encontrar o arquivo application.properties, neste arquivo o usuário deverá inserir a URL do servidor MySQL onde irá ser testada a aplicação, assim como username e senha do servidor MySQL(é necessário já ter o MYSQL Workbench instalado na máquina com pelo menos um schema para ser usado pelo BackEnd),após isso deve-se executar o arquivo 'VerzelApplication', esperar e então o servidor tomcat irá iniciar e a API vai estar rodando. Para executar o FrontEnd, deve se baixar a pasta com o nome 'testeVerzel', esta página contém a maior parte dos arquivos necessários para executar o FrontEnd porém não contém todos, nesta página está faltando a pasta node_modules que normalmente é criada ao mesmo tempo que se cria um projeto em React (que é o framework usado neste FrontEnd), ela não está lá pois é muito pesada e o github não aceitou um arquivo tão pesado assim, então eu adicionei um pequeno arquivo de testo chamado link_node_modules que contém em formato de texto um link para o meu GoogleDrive pessoal, esse link levará o usuário a esta pasta node_modules que eu armazenei no meu GoogleDrive, após isso o usuário deverá baixar esta pasta do drive e adicioná-la na pasta 'testeVerzel' onde se localiza o resto dos arquivos para o FrontEnd, com isso feito, é necessário o usuário abrir essa pasta 'testeVerzel' no VS Code, abrir um terminal na pasta e executar o código 'npm start' para iniciar a aplicação (a máquina deverá possuir node e npm já instalados para esse código funcionar), com a aplicação aberta e a API já rodando, o usuário poderá utilizar a aplicação da forma que desejar.

Para testar a aplicação será necessário ter o MySQL instalado na máquina pois o banco de dados utilizado pelo BackEnd é o MySQL mas caso seja da vontade do usuário, 
é possível utilizar outros bancos de dados, o MySQL está sendo recomendado pois é o driver de conexão do MySQL que foi instalado na aplicação BackEnd.

Cada uma das pastas vai possuir um arquivo README.md que irá explicar o que a aplicação é capaz de fazer.

Desde já agradeço pela atenção e espero que o projeto agrade a quem estiver o avaliando.
